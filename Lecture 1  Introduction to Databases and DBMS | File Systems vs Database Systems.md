# **Module 1.1 – Introduction to Databases and DBMS | File Systems vs Database Systems**

## **Learning Outcomes**

After this lecture, you will be able to:

* Understand what a Database is.
* Explain what DBMS is.
* Differentiate between File Systems and Database Systems.

---

# **1. What is a Database?**

### Definition

A **Database** is an organized collection of related data that is stored electronically and can be easily accessed, managed, and updated.

### Example

A university database stores:

* Students
* Teachers
* Courses
* Departments
* Results

**Real-Life Examples**

* YouTube → Videos, Channels, Comments
* Facebook → Users, Posts
* Amazon → Products, Customers, Orders

---

# **2. What is DBMS?**

### Definition

A **Database Management System (DBMS)** is software used to create, store, retrieve, update, and manage databases.

### Popular DBMS

* MySQL
* Oracle
* SQL Server
* PostgreSQL

---

# **3. Functions of DBMS**

* Store Data
* Retrieve Data
* Update Data
* Delete Data
* Provide Security
* Backup & Recovery

---

# **4. What is a File System?**

A **File System** stores data in separate files.

### Example

```
Students.txt
Teachers.txt
Courses.txt
```

Each file is stored separately.

---

# **Problems of File System**

* Data Redundancy (Duplicate Data)
* Data Inconsistency
* Poor Security
* Slow Searching
* Difficult Backup

---

# **5. What is a Database System?**

A **Database System** stores all related data in one centralized database managed by a DBMS.

### Advantages

* Less Data Redundancy
* Better Security
* Fast Searching
* Easy Backup
* Data Sharing
* Data Consistency

---

# **6. File System vs Database System**

| File System      | Database System      |
| ---------------- | -------------------- |
| Separate Files   | Centralized Database |
| High Redundancy  | Low Redundancy       |
| Weak Security    | Strong Security      |
| Slow Searching   | Fast Searching       |
| Difficult Backup | Easy Backup          |
